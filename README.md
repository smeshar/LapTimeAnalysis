The lightweight PyQt application is designed to work with telemetry data generated by **TinyPedals**. It analyzes `.consumption` files to help you understand your pace consistency and track your improvement over a session. The tool filters out invalid or excessively slow laps, tracks Personal Best (PB) evolution, and maps out overall pace trends.

## Features

* Minimalistic UI without unnecessary clutter.
* Dynamic threshold filtering to ignore in-lap/out-lap or crash data.
* Polynomial trend lines for overall pace and PB improvement.
* Automatic graph generation and PNG export to local directory.

## Installation

1. Clone the repository:
   ```bash
   git clone https://github.com/smeshar/LapTimeAnalysis.git
   cd lap-time-analysis
   ```

2. Install required dependencies:
    ```bash
    pip install -r requirements.txt
    ```


## Usage

### 1. Data Collection
Before using the app, ensure you have:
* **TinyPedals** running while you drive.
* Completed several laps to provide enough data for trend analysis.
* Data is automatically saved when you finish a session or exit the sim.

### 2. Locating Files
By default, TinyPedals stores the necessary telemetry in its installation folder:
`Documents\TinyPedals\deltabest\[TrackName]-[ClassName].consumption`
*(Note: The filename usually matches the track and car you were driving).*

### 3. Analysis
1. Launch `LapTimeAnalysis.exe`.
2. Click **Upload .consumption file** and navigate to the folder mentioned above.
3. Set the **Threshold time** (e.g., if your average lap is 1:35.0, set it to `96.0`).
4. Click **Generate** to view and save your progress chart.
